## 执行摘要

推荐方法是 **SRLP-GCL: Shortcut-Resistant Residual Latent Prediction for Graph Contrastive Learning**，中文名为“反捷径残差隐目标预测式图对比学习”。

一句话方法：先用教师编码器在干净图上生成目标节点隐表示，再减去邻域消息传递已经能轻易得到的平滑部分；在线编码器只能看遮蔽后的上下文图，并通过预测这个“残差隐目标”来学习节点表示，同时在残差目标空间做条件对比。

这个方向比普通“Graph-JEPA 搬到节点分类”更有论文机会，因为它直接回应近期发现：普通 GCL 的正样本对齐可能会被消息传递机制平凡化。SRLP-GCL 把正目标改造成消息传递不容易偷懒得到的残差信号，主贡献集中、第一版也能放进现有 `baselines/` 框架做小 pilot。

## 推荐方法：SRLP-GCL

### 问题锚点

- **底层问题**: 现有 GCL 在节点分类里常依赖同节点增强视图对齐，但在异配图和混合同配/异配结构中，普通正样本对齐可能不是有效学习信号。
- **必须解决的瓶颈**: 正目标不能被简单邻域平滑或同节点增强捷径提前完成；否则模型学到的是消息传递本来就会产生的低信息量一致性。
- **非目标**: 不做节点级频率路由主线，不引入 LLM 文本教师，不开新大模型栈。
- **成功条件**: 在异配图官方固定划分上超过强本地基线，并且 ablation 证明“残差目标”比全隐表示目标、普通 BGRL/GRACE 对齐更有用。

### 方法步骤

1. **教师目标构造**
   用 EMA 教师编码器 `f_bar` 在干净图上得到节点隐表示 `z_bar`。计算邻域平滑部分 `s_bar = A_norm z_bar`，然后构造目标残差：

   ```text
   r_bar(v) = normalize(z_bar(v) - stopgrad(s_bar(v)))
   ```

   直觉是：`s_bar(v)` 代表消息传递容易带来的低难度一致性；`r_bar(v)` 保留目标节点不容易由邻居平滑直接恢复的信息。

2. **上下文遮蔽**
   采样目标节点集合 `T`，遮蔽它们的输入特征，并可随机断开一部分目标节点入边，防止在线编码器直接偷看目标。在线编码器 `f` 只在遮蔽图上编码上下文。

3. **残差预测头**
   对每个目标节点 `v`，从其可见边界节点、结构统计和遮蔽后的自身 token 得到上下文向量 `c(v)`，用小 MLP 预测残差目标：

   ```text
   p(v) = g(c(v))
   L_pred = 1 - cosine(p(v), stopgrad(r_bar(v)))
   ```

4. **条件残差 InfoNCE**
   在同一 batch 或 memory bank 中选负样本，但负样本不随机选，而是从相近度数、相近局部同配率、相近残差范数的节点中取。这让任务不能靠“度数/难度/平滑性”走捷径：

   ```text
   L_ctr = -log exp(sim(p(v), r_bar(v))/tau)
           / sum_{u in C(v)} exp(sim(p(v), r_bar(u))/tau)
   ```

5. **训练与推理**
   总损失为 `L = L_pred + lambda * L_ctr + beta * L_varcov`。`L_varcov` 可复用 VICReg/Barlow 类 collapse 防护，只做稳定器，不作为论文主贡献。推理时丢掉预测头，使用编码器表示做线性评估。

### 为什么它可能新

SRLP-GCL 的增量不是“预测隐表示”本身，而是 **为图消息传递设计预测目标**。它把近期 SPGCL 暴露的问题转成方法机制：既然普通正样本会被消息传递平凡化，那就从教师目标里扣掉平滑部分，只预测剩余的残差信息。

这和几个近邻的差别如下：

- 相对 Graph-JEPA：Graph-JEPA 是图级上下文-目标子图隐表示预测；SRLP-GCL 是节点级、异配图优先，并引入残差化目标和条件对比。
- 相对 GraphMAE2：GraphMAE2 用 latent prediction 正则化特征重建；SRLP-GCL 不重建输入，也不预测全隐表示，而预测消息传递不容易得到的残差目标。
- 相对 BGRL：BGRL 预测同节点增强视图；SRLP-GCL 预测遮蔽目标节点的残差隐表示，正目标不等于同节点增强。
- 相对 ASPECT：ASPECT 学节点级谱融合策略；SRLP-GCL 不做低/高频路由，而改造自监督目标。
- 相对 SPGCL：SPGCL 从能量角度恢复正样本有效性；SRLP-GCL 直接把正样本目标改成去平滑残差，可作为互补路线。

## 最小公平 pilot

### Pilot 0：任务是否非平凡

- 数据集：Cora、Chameleon，各 1 个 seed/split。
- 对比：普通全隐表示目标预测、普通 BGRL/GRACE 目标、SRLP 残差目标。
- 成功信号：残差目标训练不 collapse；linear eval 不低于普通目标；残差目标的正负相似度分布有可分性。

### Pilot 1：异配图小表

- 数据集：Actor、Chameleon、Squirrel 的 Geom-GCN 官方 split 0。
- 对比：GRACE、GCA、BGRL、PolyGCL、GRASS、EDA-GCL 的本地对应结果。
- 成功信号：至少 2/3 数据集单 split 超过 GRACE/GCA/BGRL，且接近或超过 PolyGCL/GRASS/EDA-GCL 中的一个强基线。

### Pilot 2：正式 go/no-go

- 数据集：Actor、Chameleon、Squirrel、Cornell、Texas、Wisconsin，官方 10 固定划分。
- 主指标：Accuracy / F1Mi，报告 mean +/- std。
- Go 条件：平均比当前最好本地基线提升至少 1.0-1.5 点，或在 Chameleon/Squirrel/小 WebKB 中出现清晰互补优势。
- Kill 条件：残差目标 ablation 不如全隐目标，且没有任何异配图稳定提升。

## 推荐实现落点

第一版不要新开大框架。建议在 `baselines/` 里选一个 PyTorch GCL 基线做窄改：

1. 复用 GCN 编码器和线性评估流程。
2. 增加 EMA teacher、target masking、residual target bank、predictor head。
3. 首轮只支持 Cora、Chameleon、Actor 三个数据集。
4. 遵守 `baselines/reproduction_protocol.md`：异配图必须使用 `baselines/dataset_splits/heterophily/geom-gcn/` 官方 split。

## 排名结果

### 1. SRLP-GCL - ACTIVE

- **方法**: 预测去邻域平滑后的目标节点残差隐表示，并在条件残差目标空间做对比。
- **公平测试状态**: 尚未测试；可以 <=1 天做 pilot。
- **论文潜力**: 中高。
- **主要风险**: reviewer 可能认为残差只是高频信号，和 ASPECT/SPGCL 太近。
- **应对**: 不写成频率路由；把贡献放在“预测目标构造”和“消息传递捷径消除”。

### 2. OT 软目标预测 - SPECULATIVE

- **方法**: 多个上下文节点对多个目标节点做软分配，再预测加权目标隐表示。
- **风险**: GCL-OT 已经在异配文本属性图上很近。
- **下一步**: 只有当 SRLP-GCL pilot 较弱时，才考虑作为软正样本模块。

### 3. 环境不变预测目标 - SPECULATIVE

- **方法**: 在不同局部同配率/度数/扰动环境中预测同一目标残差的稳定部分。
- **风险**: 可能变成两个贡献叠加，首轮不要加入。

### 4. LLM 语义目标预测 - NEEDS_SETTING

- **方法**: 用文本或 LLM 表示构造目标隐表示。
- **风险**: 当前基线没有统一原始文本输入；成本和近邻工作都偏高。
