# Idea Discovery Report: DCGCL

**方向**: 重新构思一个用于节点分类提升的图对比学习方法  
**日期**: 2026-06-22  
**技能流程**: `idea-discovery` 本地执行，包含文献景观、候选发散、纸面查新、方法评审、精炼和实验计划。

## 执行摘要

本轮推荐的新主线是 **DCGCL: Disagreement-Calibrated Prototype Graph Contrastive Learning**，中文名为“分歧校准原型图对比学习”。它不再继续 SRLP 的“去捷径残差目标”路线，而是把节点分类中更直接相关的信号放到无标签原型上：同时训练或维护一个 feature-only teacher 和一个 propagation-view teacher，利用两者的原型一致与分歧来决定全局正样本、软负样本和双头表示融合。

核心判断是：SRLP 已经证明“更难预测”不等于“更有分类价值”。新方法必须让自监督目标显式接近类别边界。DCGCL 通过原型一致节点提供高置信全局正样本，通过原型分歧节点保留 feature 语义与 topology 语义的双通道，不把异配图中真实存在的语义冲突强行抹平。

推荐状态为 `ACTIVE / READY_TO_REFINE / needs pilot validation`。下一步应先做 5-epoch smoke 和 200-epoch split-0 小 pilot，而不是直接扩展 10 split 或大图。

## 推荐方法: DCGCL

### 方法做什么

1. 在同一张图上维护两个无标签 teacher。`T_x` 只看节点特征或弱增强特征，刻画 feature 语义；`T_a` 看轻量传播视图，如 GCN/PPR 后的表示，刻画 topology 语义。
2. 两个 teacher 分别产生软原型分配 `q_x(v)` 和 `q_a(v)`。若两者一致，节点进入高置信全局原型正样本池；若两者分歧，模型不强行把它们压成一个类别，而是用 feature head 和 topology head 分别学习，再由分歧门控融合。
3. 对比损失从二值 positive/negative 改为软权重：节点对的正权重由原型分配内积和置信度决定，负权重由互斥原型与边界距离决定。
4. 训练完成后丢弃 teacher 和原型头，使用融合后的 encoder 表示做标准 linear evaluation。

### 核心假设

节点分类提升来自“任务相关的无标签类别 envelope”，而不是来自更难预测的 latent residual。对同配图，一致原型提供稳定全局正样本；对异配图，feature/topology 分歧本身是有用信号，应该被校准和保留，而不是被普通 GCL 或 SRLP 式目标抹平。

### 与最近工作的差异

| 最近工作 | 重叠点 | DCGCL 的差异 |
|---|---|---|
| HEATS | 都追求全局正样本 | HEATS 学单一正样本矩阵；DCGCL 显式建模 feature teacher 与 propagation teacher 的分歧，不把所有节点压进一个 block 结构。 |
| SPGCL | 都回应正样本预对齐 | SPGCL 从 Dirichlet energy 分离传播；DCGCL 从伪类别原型一致/分歧重构 contrastive target。 |
| AFGRL / BGRL | 都可用 bootstrap teacher | DCGCL 的 teacher 不是只提供同节点目标，而是提供可解释的双原型分配和分歧门控。 |
| G-CENSOR / RGCL | 都关注任务相关视图 | DCGCL 不学习反事实视图/解释子图，而是用两个弱归纳偏置 teacher 产生 label-free class envelope。 |
| JPEB-GSSL / GraphMAE2 | 都涉及预测/聚类 | DCGCL 不做 masked reconstruction 或普通 cluster-refine，而以异配分歧校准作为主机制。 |
| ASPECT / HLCL | 都处理异配图 | DCGCL 不做频谱路由或固定滤波，而是用原型分歧决定表示融合和正负样本权重。 |

### 纸面评审

本地按 `research-review` 结构做了严格评审，未调用 sub-agent，因为当前工具策略要求用户显式允许 delegation。评审结论：

| 维度 | 分数 | 主要理由 |
|---|---:|---|
| 问题贴合度 | 8.5 | 正面回应 SRLP 的任务相关性失败。 |
| 方法具体性 | 8.0 | teacher、原型、loss、融合路径清楚，但 K 值和更新频率需在 pilot 中固定。 |
| 贡献质量 | 7.8 | 双教师分歧校准有新意，但必须严肃对比 HEATS/AFGRL/JPEB-GSSL。 |
| 可行性 | 8.2 | 可嵌入现有小图 harness，新增组件少。 |
| 验证聚焦度 | 8.0 | 首批实验可小，但必须先证明原型分歧与标签相关。 |
| 总体 | 8.1 / 10 | `READY_TO_REFINE`, 但仍需 pilot 证据。 |

主要风险：
- 原型分配可能不稳定，尤其在 Actor/Squirrel 这类噪声更强的数据上。
- 如果 `T_x` 和 `T_a` 的分歧与标签无关，方法会变成复杂 pseudo-labeling。
- 若 DCGCL 只比 BGRL/FullLatent 在一个 split 上提升，不足以扩展为论文主线。

## 结论

DCGCL 是当前最值得先做的小 pilot 路线。它避开了两个已经失败或拥挤的方向：SRLP 式残差 latent prediction 和 NFR/ASPECT 式频谱路由。它的 paper story 也更贴近节点分类：用无标签原型一致性提供全局正样本，用原型分歧保留异配图中有用的 feature/topology 冲突。

下一步不应先写大论文表，而应先实现最小组件并跑 M0/M1。如果 M1 失败，优先 kill 或降级为“原型分歧诊断”，不要再做 10 split 扩展。
