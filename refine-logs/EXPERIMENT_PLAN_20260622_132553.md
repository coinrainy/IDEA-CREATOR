## Goal

Determine whether LIFT-PROP-GCL is more than a strong baseline finding by
testing its label-free selector against validation-selected propagation, simple
graph metrics, and PROPGCL-style propagation baselines.

## Current Selector

```text
if lift(P^2X) < 0.35:
    K = 0
elif lift(P^2X) <= lift(PX) + 0.02:
    K = 1
else:
    K = 2
```

## Metrics

Each run should log:

- dataset, split protocol, K, selected K, oracle K;
- test mean/std and validation-selected K result where labels are allowed;
- `lift(PX)`, `lift(P^2X)`, `lift(P^3X)`;
- edge cosine, random-pair cosine;
- raw-vs-selected delta and selected-vs-oracle gap;
- runtime and C-grid used.

## Pass / Fail Rules

LIFT-PROP remains active only if:

1. selector v1 stays within `0.02` absolute accuracy of validation-selected K
   on most datasets;
2. raw-dominant datasets keep raw-level performance;
3. Chameleon/Squirrel propagation gains remain large;
4. novelty against PROPGCL remains defensible after direct comparison;
5. edge-lift beats or matches simpler label-free graph metrics.

R077 and R078 failed to turn LIFT-PROP into a full GCL-training method.
R079-R081 show that LIFT-gated multi-hop feature stacks are stronger than
single-depth LIFT on Cora/CiteSeer/Chameleon/Squirrel while protecting
raw-dominant graphs. This is the current strongest training-free control, but
novelty is weak because of SIGN/SGC/FAF/PROPGCL. Main-method search should
restart with LIFT-Stack as a required baseline unless the project intentionally
pivots to a selector-only paper. R082-R084 show that adding a generic hop-drop
contrastive MLP branch is not robust enough; future methods should avoid
unconstrained transformation objectives on top of LIFT-Stack. R085 shows that
existing trained BGRL/GDC/TD checkpoint embeddings are also not a stable
residual complement: they help Cora but hurt CiteSeer, Chameleon, Texas, and
Wisconsin when concatenated with LIFT-Stack. R086 shows that channel-wise
edge-lift feature weighting is also not robust: it improves Chameleon split-0
locally, but does not survive Chameleon 10-split and hurts Squirrel under ReLU
gating. R087 shows that fixed raw-propagation interaction features produce only
tiny Cora/CiteSeer variant-specific gains and fail Chameleon, so fixed-feature
micro-tuning should stop. R088 audits the fast/full C-grid discrepancy and
promotes LIFT-Portfolio as the required control: use raw for K0 graphs, use P2
when selected K2 has very low raw edge-lift, otherwise use LIFT-Stack. This is
stronger than plain LIFT-Stack on Chameleon full-grid, but still belongs to the
low-novelty fixed-propagation selector family.

R089-R094 tested three GPU-trained alternatives after LIFT-Portfolio became the
required control. EPI-BGRL failed broadly despite a Cora local gain. PAB-BGRL
improves BGRL internally on CiteSeer/Chameleon/Texas but remains far below
LIFT-Portfolio/raw controls and has high SPGCL-adjacent novelty risk. CCR-GCL
finds a strong Cora residual complement (`0.842640 -> 0.855099`) but the same
certificate admits harmful residuals on CiteSeer/Chameleon/WebKB. Do not expand
EPI/PAB/CCR to 10-split. The next plan should focus on a new mechanism or on a
principled label-free residual safety criterion before any more learned
residual training.

R095 tested that residual-safety path using existing CCR checkpoints. The
`stack_moderate_residual` rule accepts the Cora residual
(`0.842640 -> 0.851869`) and rejects CiteSeer, Chameleon, Texas, and Wisconsin,
thereby avoiding all observed split-0 damage. This is a promising diagnostic
but not a method gate: thresholds were chosen after seeing the same five
settings. The next CCR action, if any, is fixed-threshold fresh-split
validation, not another training variant.

R096 performed that validation and failed. With thresholds frozen,
`stack_moderate_residual` accepts harmful Cora residuals on splits 1 and 2
(`0.844024 -> 0.832949`, `0.854638 -> 0.850023`). It protects CiteSeer split 1,
but the Cora fresh-split failures are enough to stop CCR-SAFE. The remaining
queue was interrupted intentionally to save compute. CCR is no longer an active
main-method route.

R097 tested SBB-BGRL as a different mechanism family: low raw-cosine neighbor
messages are preserved in the forward pass but stop-gradiented during GCN
message passing. It fails the split-0 gate. Cora, CiteSeer, and Chameleon all
regress; Texas/Wisconsin improve only over weak BGRL and remain far below
LIFT/raw. Do not tune SBB quantiles or expand it.

R098 tested a fixed proxy for a view-set uncertainty/conformal direction by
adding dispersion statistics across `[X,PX,P2X,P3X]` to LIFT-Portfolio. It has
only a tiny Cora gain and hurts CiteSeer/Chameleon, so do not implement a
GPU-trained view-set dispersion objective from this proxy.

R099 tested randomized-SVD low-rank projections of LIFT-Portfolio before
implementing any learned low-rank GCL objective. It gives a small CiteSeer
gain at rank 32 (`0.725770 -> 0.731781`), but does not improve Cora and
damages Chameleon strongly (`0.699561 -> 0.651316` at the best tested rank).
Whitened coordinates are broadly worse. Treat the CiteSeer gain as a
classifier regularization diagnostic, not as a paper-level low-rank GCL route.
Final node-classification probes should remain the canonical CPU sklearn
protocol unless a GPU solver is separately validated to be numerically
equivalent.

R100-R101 tested SSA-BGRL as a different robust-optimization mechanism:
node-level BGRL losses are weighted by stability under small online-parameter
perturbations. CUDA smoke passed, but the 200-epoch split-0 gate failed as a
main route. Cora/CiteSeer improve only slightly over BGRL, Chameleon is tied,
and all completed SSA results remain far below LIFT-Portfolio. Do not run SSA
10-split or tune perturbation scale unless a new mechanism changes the signal.

R102-R103 temporarily promoted Cycle-Balance Gated LIFT, but R104-R105
downgraded it. The Chameleon gain is real (`0.685526 -> 0.725877` full-grid),
but the original sign rule degenerates to all-positive edges on Chameleon
(`edge_threshold=0.0`, `pos_ratio=1.0`). Component ablations show that
positive/no-self one-hop and two-hop propagation channels explain the gain, not
negative-edge or balanced/unbalanced signed paths. Rank-based 50/50 signs reach
only `0.685526` under fast-grid, while random/shuffled signs fail.

Decision: stop Cycle-Balance as a main-method route. Keep Positive-Path /
No-self LIFT as a required diagnostic baseline. The next plan must restart
method search and require any new candidate to add a genuinely new training or
selection principle beyond fixed propagation-channel concatenation.

R106-R107 then tested Depth-Disagreement Contrast (DDC-BGRL) as a different
mechanism family. CUDA smoke passed, but the M1 split-0 gate failed. Cora has
only a weak same-architecture improvement (`0.802953 -> 0.811260`) and remains
below standard BGRL/LIFT; CiteSeer is nearly flat (`0.677310 -> 0.678062`);
Chameleon regresses (`0.394737 -> 0.383772`) and is far below BGRL and
Positive-Path LIFT. Novelty risk is also high near BlockGCL and stage-aware
GCL. Do not run DDC 10-split or WebKB. Next method search should avoid simple
BGRL node-loss weighting families unless they include a stronger target or
representation mechanism.

R108-R109 then tested Missing-Feature Stress GCL (MFS-GCL). It uses node-wise
feature-entry dropout stress rather than the existing whole-feature-column
dropout. CUDA smoke passed, but split-0 evidence is insufficient: Cora improves
over BGRL but stays below LIFT (`0.840794` vs `0.842640`), CiteSeer is
nearly flat (`0.693839` best vs `0.693088`), and Chameleon has only a small
BGRL-internal gain while remaining far below Positive-Path
(`0.449561` vs `0.725877`). Novelty risk is high near random feature masking,
FDAGCL, and masked/adversarial feature GCL. Do not run MFS 10-split, WebKB, or
external baselines.
