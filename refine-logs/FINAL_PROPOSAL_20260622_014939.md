# Research Proposal: DCGCL

## Problem Anchor

- **Bottom-line problem**: 设计一个新的图对比学习方法，在标准节点分类实验中比现有 GCL/BGRL/SRLP-family 本地基线更稳定地提升。
- **Must-solve bottleneck**: 不能再只降低 shortcut leakage 或预测更难的 latent target；自监督目标必须保留并增强与节点类别边界相关的信号。
- **Non-goals**: 不继续 NFR-GCL 频谱路由主线；不继续 SRLP 残差预测主线；不引入需要新大模型数据栈的 LLM/text teacher 作为第一版核心。
- **Constraints**: 第一版应能嵌入 `baselines/BGRL` 或 GRACE 风格节点表示代码；异配图必须使用 Geom-GCN official fixed splits；先做小图 sequential pilot。
- **Success condition**: 在 Chameleon/Texas/Wisconsin/Actor 等异配节点分类数据上，相比 best local control 有稳定提升，并通过 ablation 证明提升来自“分歧校准原型目标”，不是参数量、训练时长或随机 pseudo-label。

## Method Thesis

**DCGCL 的核心主张**: 用 feature-only teacher 与 propagation-view teacher 的无标签原型一致和分歧来构造任务相关的软对比目标，使同配区域获得全局正样本，异配/冲突区域保留双语义表示，而不是被 same-node alignment 或残差预测抹平。

## Proposed Method

### Components

1. **Shared encoder** `f_theta`: 复用 BGRL/GRACE 的 GCN encoder。
2. **Feature teacher** `T_x`: 对 `X` 或轻增强 `X'` 编码，尽量不看边，输出 `z_x`。
3. **Propagation teacher** `T_a`: 对 `A, X` 或 PPR/GCN 轻传播视图编码，输出 `z_a`。
4. **Prototype bank** `C_x, C_a`: 每隔若干 epoch 用 EMA embedding 或 online embedding 更新，使用 KMeans 或 Sinkhorn balanced assignment。
5. **Two heads** `h_sem, h_top`: 分别学习 feature-semantics 与 topology-semantics。
6. **Disagreement gate** `g(v)`: 根据 `JS(q_x(v), q_a(v))`、assignment entropy、local compatibility 生成融合权重。

### Assignment and Disagreement

对每个节点 `v`：

```text
q_x(v) = softmax(sim(T_x(v), C_x) / tau_p)
q_a(v) = softmax(sim(T_a(v), C_a) / tau_p)
d(v)   = JS(q_x(v), q_a(v))
c(v)   = (1 - entropy(q_x)) * (1 - entropy(q_a)) * exp(-d(v))
```

- `d(v)` 小且 `c(v)` 高：节点属于 agreement pool。
- `d(v)` 大：节点属于 disagreement pool，不强制单原型。

### Loss

**1. Agreement prototype contrast**

对高置信一致节点，用融合原型 `q_star = normalize(q_x * q_a)` 构造软正样本：

```text
W_ij = c_i * c_j * dot(q_star_i, q_star_j)
L_softNCE = - sum_i log sum_j W_ij exp(sim(p_i, t_j)/tau)
            / sum_j exp(sim(p_i, t_j)/tau)
```

**2. Disagreement dual-head learning**

对分歧节点，不让一个 head 同时解释两种语义：

```text
L_dual = CE(p_sem(v), stopgrad(q_x(v))) + CE(p_top(v), stopgrad(q_a(v)))
```

并约束两个 head 在分歧节点上不过度坍缩：

```text
L_orth = |cos(h_sem(v), h_top(v))|  for high-disagreement nodes
```

**3. Stability loss**

保留轻量 BGRL-style same-node bootstrap 作为稳定器：

```text
L_stab = 1 - cosine(pred(z_view1(v)), stopgrad(z_view2(v)))
```

总损失：

```text
L = L_softNCE + lambda_dual L_dual + lambda_orth L_orth
    + lambda_stab L_stab + lambda_bal L_balance
```

`L_balance` 是 prototype entropy / Sinkhorn balance，防止所有节点落入少数原型。

### Inference

训练结束后丢弃 teacher、prototype bank 和 projection heads。使用：

```text
z_final(v) = (1 - g(v)) * h_sem(v) + g(v) * h_top(v)
```

或拼接 `[h_sem(v), h_top(v), g(v)]` 做 linear evaluation。第一版优先使用加权和，避免线性 probe 参数量膨胀。

## Why This Is the Smallest Adequate Route

- 它只新增两个 teacher assignment、一个 prototype bank、一个 soft contrast loss 和一个双头/gate，不需要新数据、文本、LLM 或大图预训练。
- 它直接回应 SRLP 失败原因：目标必须任务相关，而不是单纯反捷径。
- 它避免 NFR/ASPECT 的频谱拥挤路线，也避免 GraphMAE/JEPA 的普通 latent prediction 路线。
- 它的关键 ablation 很清楚：去掉 dual teacher、去掉 disagreement gate、把 soft prototype 改成 HEATS-like single affinity，就能检验主贡献。

## Closest Work and Positioning

- **HEATS**: 提出全局 adaptive positive sampling，最近且必须强 baseline。DCGCL 的差异是把单一 affinity 改成双教师分歧校准，不假设所有节点都应进入同一个 block partition。
- **SPGCL**: 处理消息传递导致的 positive pre-alignment。DCGCL 不做 Dirichlet energy propagation，而把 positive 的可靠性转成 prototype agreement。
- **AFGRL/BGRL**: 提供 bootstrap 和 global positive 先例。DCGCL 的 teacher 输出可解释原型，不只是 target embedding。
- **G-CENSOR/RGCL/Community-Invariant GCL**: 提供任务相关视图/不变性先例。DCGCL 的主线不是 learned view generator，而是 label-free disagreement-calibrated prototype target。
- **JPEB-GSSL/GraphMAE2**: cluster/predictive graph SSL 近邻。DCGCL 不做 masked reconstruction 或 generic cluster refine，必须强调异配分歧校准。

## Claim-Driven Validation Sketch

### Claim 1: 原型分歧校准比普通 same-node alignment 和残差预测更贴近节点分类

- **Minimal experiment**: Chameleon/Texas/Wisconsin/Cora split-0，200 epoch。
- **Baselines**: BGRL, FullLatent, ZPZ, SRLP-Aux, DCGCL。
- **Ablations**: single teacher prototype, no disagreement gate, hard prototype instead of soft prototype。
- **Metrics**: `valid@best`, `test@best`, prototype NMI/ARI to labels for diagnostics only, assignment entropy, collapse flag。
- **Expected evidence**: DCGCL 在至少 3/4 数据集不低于 best control，并在 2 个异配图有清晰提升。

### Claim 2: 分歧节点需要双头保留，而不是强制单一 pseudo-class

- **Minimal experiment**: 对 agreement/disagreement 节点分桶报告 linear probe accuracy 或 probe confidence。
- **Baselines**: DCGCL full, no dual-head, no orthogonality, single-fusion。
- **Metrics**: high-disagreement bucket accuracy, global accuracy, gate-label mutual information。
- **Expected evidence**: no dual-head 主要伤害高分歧节点，证明主机制不是普通 pseudo-label contrast。

## Failure Modes

| Failure | Detection | Response |
|---|---|---|
| Prototype collapse | prototype usage entropy 低，最大原型占比过高 | 增强 Sinkhorn balance，降低 K，延长 warmup。 |
| Teacher disagreement 与标签无关 | `d(v)` 与错误率/label entropy 无相关 | Kill 主线，不做 10 split。 |
| 只在同配图提升 | Cora/CiteSeer 提升但 heterophily 持平 | 降级为 homophily 方法，不符合当前目标。 |
| 参数量解释提升 | no-dual-head 接近 full | 删除双头，转向 simpler soft prototype contrast。 |

## Compute Estimate

- M0 smoke: < 0.5 GPU-hour。
- M1 split-0 pilot: 约 3-6 GPU-hours sequential。
- M2 10-split gate: 约 25-45 GPU-hours，只有 M1 通过后再跑。

## Verdict

`READY_TO_PILOT`。方法具体、可实现、与 SRLP 失败教训对齐，但没有实验证据前不得声称提升。
