**Candidate**: NPG-GCL, Nontrivial Positive Gain Graph Contrastive Learning

## Problem Anchor

Recent graph contrastive learning papers increasingly challenge whether positive alignment is still informative after message passing has already made many node representations easy to align. A 2026-level method should not simply mine more positives; it should identify when positive alignment provides nontrivial learning signal beyond a cheap propagation baseline.

## Method Thesis

NPG-GCL trains a BGRL-style encoder but downweights node-wise positive alignment when the online representation is already no harder to align than a simple message-passing prealignment baseline. It keeps the bootstrap objective, but makes the loss focus on nodes whose positive alignment remains nontrivial.

## Minimal Formulation

Let `q_i` be the online prediction and `y_i` be the stop-gradient target for node `i`.

The ordinary BGRL node loss is:

```text
ell_i = 2 - cos(q_i^1, y_i^2) - cos(q_i^2, y_i^1)
```

NPG computes a cheap prealignment baseline from normalized propagated raw features:

```text
u = normalize(X W_anchor)
p = normalize(P u)
pre_i = 1 - cos(u_i, p_i)
```

where `P` is the normalized adjacency with self-loops and `W_anchor` is a frozen random projection or identity-compatible linear projection used only to match dimensions.

The nontrivial-gain weight is:

```text
gain_i = relu(pre_i - detached_alignment_i + margin)
w_i = normalize(clamp(gain_i, w_min, w_max))
L_npg = mean_i w_i * ell_i
```

Intuition: if a node is already easy under cheap propagation, extra positive alignment is likely trivial; if the learned alignment is not explained by propagation, the objective is allowed to spend capacity there.

| Claim | Required evidence |
|---|---|
