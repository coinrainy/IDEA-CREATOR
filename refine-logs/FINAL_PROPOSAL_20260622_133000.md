# Final Proposal Draft: LIFT-PROP-GCL

**Date**: 2026-06-22 12:50  
**Status**: `SELECTOR_BASELINE_NOT_MAIN_METHOD`, not paper-ready  
**Candidate**: LIFT-PROP-GCL, Label-free edge-LIFT gated propagation for graph contrastive learning

## Problem Anchor

Recent graph contrastive learning work shows that simple propagation can rival
or beat complex learned GCL objectives. The unresolved practical problem is not
only how to design stronger propagation, but how to decide when propagation
should be used at all. In the current experiments, propagation is strongly
helpful on Chameleon/Squirrel and harmful on WebKB/Actor-style raw-dominant
graphs.

## Method Thesis

LIFT-PROP-GCL uses a label-free reliability statistic to route between raw
features and propagated features before contrastive training or evaluation. It
keeps the contribution narrow:

> edge-vs-random cosine lift predicts whether propagation creates useful
> graph-aligned structure or merely oversmooths/damages raw-feature signal.

## Selector v1

Let `P` be the normalized adjacency with self-loops and let `X_k = P^k X`.
Define:

```text
lift(X_k) = mean_(i,j in E) cos(X_k[i], X_k[j])
          - mean_(u,v random) cos(X_k[u], X_k[v])
```

Selector v1:

```text
if lift(X_2) < 0.35:
    select K = 0
else if lift(X_2) <= lift(X_1) + 0.02:
    select K = 1
else:
    select K = 2
```

The selected representation is `X_K`. In the GCL framing, this is the
reliability gate for a propagation-contrastive view: propagation is enabled
only when the graph produces nontrivial edge-lift over random-pair alignment.

## Current Evidence

Selector v1 matches the observed oracle K on 7/8 current datasets/settings:

| Dataset / setting | Selected K | Result | Oracle K | Status |
|---|---:|---:|---:|---|
| Cora split-0 | 2 | 0.826488 | 3 | near miss |
| CiteSeer split-0 | 2 | 0.709617 | 2 | hit |
| Chameleon 10-split | 2 | 0.685746 +/- 0.021187 | 2 | hit |
| Texas 10-split | 0 | 0.829730 +/- 0.051042 | 0 | hit |
| Wisconsin 10-split | 0 | 0.839216 +/- 0.043157 | 0 | hit |
| Cornell 10-split | 0 | 0.816216 +/- 0.059487 | 0 | hit |
| Actor 10-split fast grid | 0 | 0.347566 | 0 | hit |
| Squirrel 10-split fast grid | 1 | 0.520365 | 1 | hit |

Novelty check score is `5.5/10`, `PROCEED_WITH_CAUTION`. The closest prior is
PROPGCL. The main differentiator must be raw-vs-propagation abstention and
label-free K selection, not propagation itself.

PROPGCL-facing first-pass comparison is positive for the selector claim:
LIFT-PROP v1 reaches mean oracle gap `0.001096` across the eight observed
settings, while the PROPGCL Appendix-E reported-step heuristic has mean oracle
gap `0.011052`. This does **not** beat learned PROPGCL (`PROP-GRACE` /
`PROP-DGI`); it only establishes a label-free selector delta over fixed
PROP-step heuristics.

R077/R078 attempted to turn LIFT-PROP into a fuller GCL method and failed.
Edge-NCE learned propagation coefficients improved only Cora but damaged
CiteSeer and Chameleon; node-wise LIFT routing improved only Cora and hurt
Chameleon/Texas/Wisconsin. The useful contribution is therefore the global
selector itself, not learned coefficient training or per-node K routing.

## Required Positioning

Do not claim:

- first use of propagation in GCL;
- first feature-node graph construction;
- first graph/raw dual-view method;
- first adaptive spectral or filtering GCL method.

Claim only if supported:

- a simple zero-label edge-lift statistic predicts when graph propagation helps
  versus hurts;
- reliability-gated propagation protects raw-dominant heterophily datasets
  while retaining Chameleon/Squirrel gains;
- the gate complements PROPGCL-style propagation by removing the need for
  validation-label propagation-depth selection in the raw-vs-propagation
  decision.
