## Method

FTDR routes each node's bootstrap target between a topology teacher and a feature-only teacher:

```text
z_g(v) = stopgrad(target_encoder(G, X)_v)
z_f(v) = stopgrad(target_encoder(I, X)_v)
d_v = 1 - cosine(z_g(v), z_f(v))
w_v = sigmoid((d_v - tau_d) / temp) * w_max
y_v = normalize((1 - w_v) * z_g(v) + w_v * z_f(v))
```

The goal was to avoid both SRLP residual targets and DCGCL global prototype positives while retaining a BGRL-like fallback when feature/topology teachers agree.

## Decision

FTDR is stable but not positive. It nearly preserves Cora and ties Texas/Wisconsin, but it does not improve heterophily and still hurts Chameleon. It should not proceed to 10-split expansion.

The repeated pattern across SRLP, DCGCL, and FTDR is that label-free routing/target decomposition can produce isolated dataset positives but has not produced a robust method claim. The next discovery round should diverge away from latent-target routing and prototype/teacher routing.
